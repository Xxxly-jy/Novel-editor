# 小说记忆工作台

一个本地优先的小说创作原型。它把章节、人物卡、大纲、世界观和长期记忆放在同一个写作界面里，并支持接入 OpenAI 兼容格式的 API。

## 本地优先原则

- 工具不提供服务器，不托管用户小说内容。
- 作品、章节、设定、记忆库和 API 设置默认只保存在当前浏览器本地。
- 点击“生成”前需要勾选发送确认；只有用户主动生成时，当前上下文才会发送给用户自己配置的 API 服务商。
- “复制上下文”只复制到本机剪贴板，不会联网。
- 本地备份默认不包含 API Key。

## 使用方式

直接用浏览器打开 `index.html`。

数据会保存在当前浏览器的 `localStorage` 中，包括正文、设定和 API 配置。

## API 设置

在右侧切到“设置”，填写：

- 接入预设：可选 Ollama、LM Studio、OpenRouter、Groq、DeepSeek 或自定义
- 接口地址：例如 `https://api.openai.com/v1`
- API Key：你自己的 key
- 模型名称：例如 `gpt-4.1`、`deepseek-chat`、`qwen-plus`
- 温度：建议 `0.7` 到 `1.0`
- 最大输出：建议 `1000` 到 `3000`

### 高级自定义接口

设置页里的“高级接口设置”可以自定义：

- 请求路径：默认 `/chat/completions`
- 鉴权 Header：默认 `Authorization`
- Key 前缀：默认 `Bearer`
- 额外 Headers JSON：例如 `{"HTTP-Referer":"https://your-site.example","X-Title":"Novel Studio"}`

如果“请求路径”留空，工具会把“接口地址”当作完整请求地址使用。这样可以接入不完全按照 `/v1/chat/completions` 组织路径的服务。

### 免费 / 低成本接入建议

- Ollama：本地运行，通常不需要 API Key。预设接口为 `http://localhost:11434/v1`。
- LM Studio：本地运行，通常不需要 API Key。预设接口为 `http://localhost:1234/v1`。
- OpenRouter：在线服务，需要用户自己的 API Key，可在服务商后台选择当前可用的免费模型。
- Groq：在线服务，需要用户自己的 API Key，免费额度和模型列表以账号后台为准。
- DeepSeek：在线服务，需要用户自己的 API Key，预设使用 `https://api.deepseek.com` 和 `deepseek-v4-flash`。

浏览器直连部分 API 时可能遇到 CORS 限制。如果生成失败，可以先用“复制上下文”把完整上下文复制到其他 AI 工具里。后续做 Electron 桌面版时，可以通过本地代理解决这个问题。

## 当前功能

- 章节创建、切换、删除
- 章节状态：草稿、修订、完成
- 章节目标字数和进度显示
- 专注模式，隐藏两侧面板进入纯写作状态
- 章节快照，保存/恢复当前章节版本
- 正文自动保存
- 章节摘要，帮助 AI 快速理解前文
- 全局搜索正文、人物、大纲、世界观和长期记忆
- 人物卡、大纲、世界观、长期记忆
- 从当前章节生成“长期记忆提取提示”
- 把 AI 提取结果自动拆成长期记忆卡
- 本地 JSON 备份与导入
- 生成前 API 发送确认
- 清除本地 API Key
- AI 接入预设：Ollama、LM Studio、OpenRouter、Groq、DeepSeek、自定义接口
- AI 续写、润色、扩写、检查矛盾、生成后续大纲
- 复制完整 AI 上下文
- 导出 TXT / MD

## 记忆提取流程

1. 写完一章后，在章节标题下填写摘要。
2. 切到“长期记忆”，点击“从本章提取”。
3. 如果已经配置 API，点击“生成”；如果没有配置 API，就复制输出区的提示词到其他 AI 工具。
4. 把 AI 返回结果放在输出区，点击“保存为记忆”。
